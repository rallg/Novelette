Interior Images

Your book interior may have images. There are several rules:

  • No color. Period.
  • Line art (black/white only, no gray or anit-aliasing) must be 1-bit per pixel. Standard resolution is 600 pixels per inch (ppi). If your image has lower resolution, then it is better to use grayscale.
  • Grayscale images must be single-channel shades of black ink. That is, 8-bits per pixel. Only 300ppi resolution is allowed.
  • Do not convert images into "print screen" dots. That will be done by the P.O.D. Service, if necessary.
  • Image metadata must state resolution, must not name or include a color profile, must not contain EXIF or similar data.

To ensure that all of the above specifications are met, Novelette comes with several scripts. These are run in command-line terminal (Windows Command, Linux BASH). Each image must be processed by the appropriate script. The processed image will have an internal marker, informing Novelette that the image can be used.

When you place an image, Novelette inspects its raw digital data, and looks for the scripted marker. If not there, the image is rejected.

Why do this? PDF/X-1a:2001 places tight restrictions on images. This ensures compliance. Without the restrictions, images would still print well on a home or office printer, because your own printer will guess what you mean. But files with non-compliant images might be rejected by some P.O.D. Services, because they are not in the business of guessing.

Maximum Image Dimensions

Novelette does not scale or crop images to fit. Your images must be at the exact resolution (pixels per inch) and number of pixels (width, height) that they will be used.

Each template has maximum image dimensions. If you place an image that exceeds those values, Novelette throws an error.

\image[placement]{filename}

The placement is optional. Default is h. Allowable placement is one of:

  • h   Place it "here" if possible, immediately after the current paragraph finishes. But if it does not fit, then place it at the top of the following page.
  • t   Place it at the "top" of the following page. But you cannot place it on the same page, above where the code is written.
  • b   Place it at the "bottom" of the current page, if possible. But if it does not fit, place it at the bottom of the following page.

The filename must include its file extension. The extension must be png, jpg, jpeg, or their capitalized forms. Paths such as images/filename are allowed. No spaces anywhere. Case-sensitive.

The image will be placed as a block element, horizontally centered in the printable area. If you need to shift it left or right, re-work the image in your graphics program, then re-process the image with script.

No border is added. If it needs a border, make it part of the image. No caption is added. If it needs a caption, make it part of the image.

Novelette will add a variable amount of space, above and below the image. This automatically increases the occupied area to an integral number of text lines.
